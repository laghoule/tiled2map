# tiled2map

`tiled2map` is a command-line tool written in Go that converts [Tiled](https://www.mapeditor.org/) map files (JSON format) into binary map data and assembly source files ready to be used in retro-style game development (e.g. targeting 8-bit or 16-bit platforms).

## Overview

The tool takes a Tiled map exported as JSON and produces:

- **`<prefix>.png`** — A master tile atlas image containing all unique tiles used in the map, packed vertically.
- **`<prefix>.til`** — A binary tile atlas file (raw indexed pixel data) with a small header, suitable for direct loading on target hardware.
- **`<prefix>-bg-world.map`** — Binary background layer map data, organized by scenes.
- **`<prefix>-fg-world.map`** — Binary foreground layer map data, organized by scenes.
- **`<prefix>-refs.inc`** — An assembly include file listing per-tile attribute bytes (`TILES_PROPS`).
- **`<prefix>-scene.inc`** — An assembly include file describing each scene with its neighbors and music reference.

## Pipeline

```
Tiled JSON map
      │
      ▼
 tiled package          ← Parse map, layers, tilesets, GIDs
      │
      ├─► atlas package ← Build master tile atlas (PNG + TIL)
      │
      └─► asm package   ← Generate binary maps + ASM include files
```

## Map Requirements

The Tiled map **must** be exported in JSON format and contain the following named layers:

| Layer name | Type         | Description                           |
| ---------- | ------------ | ------------------------------------- |
| `bg`       | Tile layer   | Background tiles (required)           |
| `fg`       | Tile layer   | Foreground / sprite layer (required)  |
| `bound`    | Object layer | Collision/boundary objects (optional) |

The map must have non-zero dimensions, and its width and height must be exact multiples of the scene dimension.

All tilesets referenced by the map must use **paletted PNG images** sharing the same color palette (the palette of the first tileset image is used as the master palette).

Tiles can carry a custom integer property named **`attr`** in Tiled. This value is written as a binary byte in the generated `TILES_PROPS` table in the refs include file. Tiles without this property default to `00000000b`.

## Usage

```
tiled2map -map <path/to/map.json> [-dimension <WxH>] [-fileprefix <prefix>]
```

### Flags

| Flag          | Default      | Description                                |
| ------------- | ------------ | ------------------------------------------ |
| `-map`        | _(required)_ | Path to the Tiled map file in JSON format  |
| `-dimension`  | `20x11`      | Width × height of each scene (in tiles)    |
| `-fileprefix` | `master`     | Prefix used for all generated output files |

### Example

```
tiled2map -map map.json -dimension 20x11 -fileprefix master
```

This will generate:

```
master.png
master.til
master-bg-world.map
master-fg-world.map
master-refs.inc
master-scene.inc
```

## Output File Formats

### `.til` — Binary Tile Atlas

| Offset | Size    | Description            |
| ------ | ------- | ---------------------- |
| 0      | 1 byte  | Tile width (pixels)    |
| 1      | 1 byte  | Tile height (pixels)   |
| 2      | 1 byte  | Number of tiles        |
| 3+     | N bytes | Raw indexed pixel data |

Tiles are stored sequentially, one after another, sorted by GID order.

### `.map` — Binary Map Data

| Offset | Size    | Description             |
| ------ | ------- | ----------------------- |
| 0      | 1 byte  | Scene width (in tiles)  |
| 1      | 1 byte  | Scene height (in tiles) |
| 2+     | N bytes | Local tile IDs (uint8)  |

Map data is organized **scene by scene** (left-to-right, top-to-bottom). Within each scene, tiles are stored row by row. The local tile ID is the index of the tile in the master atlas (0-based, sorted by GID).

### `-refs.inc` — Assembly Tile Properties

```asm
; GENERATED BY tiled2map (https://github.com/laghoule/tiled2map)
; DO NOT EDIT

TILES_PROPS LABEL BYTE
  DB 00000000b ; Index: 0 (GID:   6 Source: set_0.png)
  DB 00000001b ; Index: 1 (GID:  24 Source: basictiles.png)
  ...
```

### `-scene.inc` — Assembly Scene Descriptors

Each scene entry uses a `SCENE` struct with a shared map buffer offset (used for both background and foreground), a music reference, and neighboring scene references (North, South, East, West). A value of `0` means no neighbor in that direction.

```asm
; GENERATED BY tiled2map (https://github.com/laghoule/tiled2map)
; DO NOT EDIT

; --- Scene # 0 ---
SCENE_0_0 SCENE < \
  <OFFSET world_bg_buffer + 2, 0>, \
  <OFFSET world_fg_buffer + 2, 1>, \
  OFFSET MUSIC_0_0, \
  0, \
  OFFSET SCENE_0_1, \
  OFFSET SCENE_1_0, \
  0  \
>
```

The scene offset formula is: `((y * numScenesX) + x) * sceneTiles + headerSize`, where `sceneTiles = sceneWidth × sceneHeight` and `headerSize = 2`.

## Building

### Prerequisites

- Go 1.26 or later

### Build from source

```
git clone https://github.com/laghoule/tiled2map.git
cd tiled2map
go build -o tiled2map ./cmd/main.go
```

### Build with version information

```
go build \
  -ldflags="-X 'main.version=1.0.0' -X 'main.gitCommit=$(git rev-parse --short HEAD)'" \
  -o tiled2map ./cmd/main.go
```

### Docker

A `Dockerfile` is provided to build a minimal container image based on `scratch`:

```
docker build \
  --build-arg VERSION=1.0.0 \
  --build-arg GIT_COMMIT=$(git rev-parse --short HEAD) \
  -t tiled2map .
```

Run with Docker:

```
docker run --rm -v $(pwd):/data tiled2map -map /data/map.json -fileprefix /data/master
```
